## 1. Introduction

이 문서는 'KTX-AutoReserve'라는 코레일 KTX 승차권 자동 예매 시스템에 대한 분석(Analysis) 보고서입니다.

KTX-AutoReserve는 사용자가 단일 API 요청만으로 코레일 웹사이트에 로그인하고, 원하는 조건의 KTX 열차를 자동으로 탐색하여 예약을 시도하는 Spring Boot 기반의 웹 자동화 시스템입니다. Selenium WebDriver를 통해 브라우저 UI를 제어하며, 로그인부터 예약 버튼 클릭까지의 전 과정을 일관된 로직으로 수행합니다.

이 보고서에서는 KTX-AutoReserve 프로젝트가 가진 다음 특징들을 중심으로 분석 내용을 담고 있습니다.

**usefulness**: 명절·연휴·출퇴근 시간대 등 경쟁이 치열한 KTX 좌석 예매 환경에서, 사용자가 반복적으로 화면을 새로고침하며 수동으로 클릭해야 하는 부담을 없애고 일관된 자동화 로직으로 예약 성공률과 신속성을 높이는 데 실질적인 도움을 제공합니다.

**significance**: Selenium WebDriver 기반의 브라우저 UI 자동화와 Spring Boot REST API를 결합하여, 단일 HTTP 요청 하나만으로 로그인~예약 시도까지의 전 과정을 처리하는 차별화된 구조를 갖습니다. 또한 AWS SNS 연동을 통해 예약 성공 즉시 SMS 알림을 전송하는 기능도 포함합니다.

**expandability**: 현재 KTX 예매에 집중되어 있으나, 출발역·도착역·날짜·시간 조건을 파라미터화한 구조 덕분에 다양한 노선·열차 유형으로 확장하거나, 복수 예약 시도·스케줄 기반 자동 실행 등으로 기능을 발전시킬 수 있는 잠재력을 가집니다.

이어서 보고서에서는 KTX-AutoReserve 시스템의 구체적인 요구사항 및 구조를 파악하기 위한 Use Case 분석, Domain 분석, 그리고 사용자 인터페이스 프로토타입 소개가 이어집니다.

## 3. Domain analysis

아래는 KTX-AutoReserve 시스템의 Domain Analysis에서 나오는 Class들의 관계를 나타낸 것이다.

                           │ starts
          ┌────────────────┼────────────────┐
          ▼                ▼                ▼
   TicketController  GlobalException   SwaggerConfig

                    ▼            ▼
            TicketReservation  SmsSender

### 애플리케이션 진입점

**KorailReserveApplication**: Spring Boot 애플리케이션의 진입점 클래스이다. `main()` 메서드에서 `SpringApplication.run()`을 호출하여 서버를 기동하고, 컨트롤러·서비스·설정 빈을 로드하여 API를 제공한다.

### 웹 계층

**TicketController**: HTTP 요청을 수신하는 REST 컨트롤러이다. `/ticket/reserve` 엔드포인트를 통해 사용자 요청을 받고, 쿼리 파라미터의 형식·범위·시간 순서를 검증한 후 TicketService를 호출한다. 잘못된 요청을 조기에 차단하여 정상 입력만 서비스 계층으로 전달한다.

**GlobalExceptionHandler**: `@RestControllerAdvice`로 구현된 전역 예외 처리 클래스이다. 누락 파라미터(`MissingServletRequestParameterException`)와 타입 오류(`MethodArgumentTypeMismatchException`) 예외를 처리하여 Whitelabel 오류 대신 의미 있는 400 응답을 제공한다.

**SwaggerConfig**: OpenAPI 문서 메타데이터를 설정하는 클래스이다. OpenAPI 빈을 등록하여 Swagger UI에서 API 테스트와 문서 확인이 가능하도록 한다. 앱 시작 시 초기화된다.

### 자동화 오케스트레이션

**TicketService**: 예약 자동화 전체 시나리오를 오케스트레이션하는 핵심 서비스 클래스이다. WebDriverFactory → KorailLoginService → 화면이동 → SearchCriteriaService → TicketReservationService 순서로 자동화 흐름을 제어한다. 요청 단위로 실행되며 종료 시 드라이버를 정리한다.

### 브라우저 제어

**WebDriverFactory**: 브라우저 드라이버 생성을 전담하는 클래스이다. Chrome/Safari 선택, 헤드리스 옵션, 창 크기 초기화 등을 담당하여 브라우저 실행 설정 변경 시 영향 범위를 최소화한다.

**KorailLoginService**: 코레일 로그인 UI 흐름을 처리하는 전용 클래스이다. 로그인 페이지 진입, 공지 팝업 닫기, 아이디/비밀번호 입력, 로그인 버튼 클릭을 순서대로 수행하며, 예약 절차 초반 1회 실행된다.

**SearchCriteriaService**: 조회 조건 입력 로직을 담당하는 클래스이다. 출발역/도착역 팝업 선택, 날짜 팝업 선택, 시간 슬라이더 조작을 수행하며, 재사용성과 테스트 용이성을 위해 TicketService로부터 분리되었다.

**TicketReservationService**: 열차 목록 판별 및 예약 클릭을 담당하는 클래스이다. 조회 결과 목록을 순회하며 출발 시각(Time Window)과 KTX 여부(KTX Filter)를 필터링하고, 조건에 맞는 열차의 예약 버튼을 클릭한다. 조회 루프 내에서 반복 실행된다.

### 알림

**SmsSender**: 예약 성공 시 SMS 알림을 전송하는 클래스이다. AWS SNS 클라이언트를 사용하여 메시지를 publish하며, 성공 알림 로직이 활성화된 경우에만 호출된다.

## 4. User Interface prototype

본 시스템은 REST API 기반 백엔드 자동화 시스템으로, 별도의 GUI 화면 대신 Swagger UI와 HTTP API 인터페이스를 통해 사용자와 상호작용한다.

### 4.1 Swagger UI

시스템이 실행되면 `/swagger-ui` 에서 Swagger UI를 통해 API 명세를 확인하고 테스트할 수 있다.

```
┌─────────────────────────────────────────────────────────┐
│  KTX-AutoReserve API                                    │
│  Korail KTX 자동 예매 API                                │
├─────────────────────────────────────────────────────────┤
│  GET  /ticket/reserve   티켓 예약 자동화 실행           │
│                                                         │
│  Parameters                                             │
│  ┌──────────────────────────────────────────────────┐   │
│  │ departure   string  required  출발역 (예: 동대구) │   │
│  │ arrival     string  required  도착역 (예: 서울)   │   │
│  │ date        string  required  날짜 (yyyyMMdd)     │   │
│  │ timeStart   integer required  탐색 시작 시간 (0~23)│  │
│  │ timeEnd     integer required  탐색 종료 시간 (0~23)│  │
│  └──────────────────────────────────────────────────┘   │
│                                          [Execute]       │
└─────────────────────────────────────────────────────────┘
```

### 4.2 예약 API 요청

사용자는 아래와 같은 형식으로 HTTP GET 요청을 전송하여 예약을 시도한다.

**Endpoint**: `GET /ticket/reserve`

**Query Parameters**:

| Parameter | Type | Required | Description | Example |
|---|---|---|---|---|
| departure | String | Y | 출발역 | 동대구 |
| arrival | String | Y | 도착역 | 서울 |
| date | String | Y | 출발 날짜 (yyyyMMdd) | 20260501 |
| timeStart | int | Y | 탐색 시작 시간 (0~23) | 6 |
| timeEnd | int | Y | 탐색 종료 시간 (0~23) | 10 |

**요청 예시**:
```
GET /ticket/reserve?departure=동대구&arrival=서울&date=20260501&timeStart=6&timeEnd=10
```

### 4.3 응답 예시

**예약 성공 (200 OK)**:
```json
{
  "message": "예약이 성공적으로 완료되었습니다."
}
```

**파라미터 오류 (400 Bad Request)**:
```json
{
  "message": "필수 파라미터가 누락되었습니다: departure"
}
```

**타입 오류 (400 Bad Request)**:
```json
{
  "message": "파라미터 타입이 올바르지 않습니다: timeStart"
}
```

**탐색 실패**:
```json
{
  "message": "조건에 맞는 예약 가능한 열차를 찾지 못했습니다."
}
```

### 4.4 자동화 실행 흐름

아래는 API 요청 수신 후 시스템 내부에서 실행되는 자동화 흐름이다.

```
[사용자 GET 요청]
       ↓
TicketController
  ├─ 파라미터 검증 실패 → GlobalExceptionHandler → 400 응답
  └─ 파라미터 검증 성공
       ↓
 TicketService
       ↓
 WebDriverFactory → 브라우저 실행
       ↓
 KorailLoginService → 코레일 로그인 (팝업 처리 포함)
       ↓
 화면 이동 → 일반 승차권 예매 페이지
       ↓
 SearchCriteriaService → 출발역 / 도착역 / 날짜 / 시간 입력 및 조회
       ↓
 TicketReservationService → 열차 목록 탐색
  ├─ 조건 불일치 → Polling 후 재조회 ────────────┐
  └─ 조건 일치 → 예약 버튼 클릭                 (반복)
       ├─ 예약 실패 → 재탐색으로 복귀 ────────────┘
       └─ 예약 성공
            ↓
       SmsSender (전화번호 파라미터 존재 시)
            ↓
       WebDriver 세션 종료
            ↓
      [결과 응답 반환]
```

## 5. Glossary

| Terms | Description |
|---|---|
| Reservation API | 티켓 예약 자동화 요청 엔드포인트(`/ticket/reserve`) |
| WebDriver | 브라우저를 자동 제어하는 Selenium 인터페이스 |
| Explicit Wait | 특정 조건이 만족될 때까지 대기하는 동기화 방식 |
| Popup Overlay | 메인 화면 클릭을 가로채는 모달/오버레이 계층 |
| Polling | 일정 간격으로 결과를 재조회하는 반복 전략 |
| KTX Filter | 열차 목록에서 KTX만 선별하는 조건 |
| Time Window | 사용자가 지정한 출발 시각 허용 범위 |
| Exception Handler | 예외를 HTTP 응답으로 변환하는 처리 계층 |
| OpenAPI | REST API 문서화 표준 (Swagger UI 기반) |
| SNS Notification | AWS SNS를 통한 문자 알림 전송 기능 |
| Headless Mode | 화면 출력 없이 브라우저를 백그라운드에서 실행하는 방식 |
| Orchestration | 여러 서비스의 실행 순서를 조율하는 제어 방식 |
